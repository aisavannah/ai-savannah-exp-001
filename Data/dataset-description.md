# EXP-001 — Dataset Description

**AI Savannah Governance Lab**

---

## Overview

EXP-001 does not use a training dataset. It uses a **prompt dataset** and a **ground truth regulatory corpus**. This file describes both.

---

## Prompt Dataset

**File:** `prompts/prompt-set-v1.md`
**Total prompts:** 16 (P-001 through P-016)
**Categories:**

| Category | Count | Description |
|---|---|---|
| A — Accurate regulatory questions | 5 | P-001 to P-005 |
| B — Fictional regulatory references | 5 | P-006 to P-010 |
| C — Foreign framework tests | 3 | P-011 to P-013 |
| D — Confidence and uncertainty tests | 3 | P-014 to P-016 |

**Runs per prompt per model:** 3
**Total models:** 4 (GPT-4o, Gemini 1.5 Pro, Mistral-7B, Falcon-7B)
**Total individual runs:** 16 prompts × 3 runs × 4 models = 192 runs

## Ground Truth Regulatory Corpus

**File:** `data/source-documents-list.md`

The ground truth corpus consists of five primary source documents. All are publicly available. All were downloaded and saved before the experiment was run. Model outputs are fact-checked against these documents only — not against secondary commentary, legal blogs, or AI-generated summaries.

| ID | Document | Year | Issuing Body |
|---|---|---|---|
| DOC-001 | CBK Digital Credit Provider Regulations | 2022 | Central Bank of Kenya |
| DOC-002 | Kenya Data Protection Act | 2019 | Republic of Kenya |
| DOC-003 | Proceeds of Crime and Anti-Money Laundering Act (POCAMLA) | 2009 (amended) | Republic of Kenya |
| DOC-004 | CBK Prudential Guidelines — AML/CFT | 2013 (revised) | Central Bank of Kenya |
| DOC-005 | Kenya Artificial Intelligence Bill | 2026 | Republic of Kenya — Senate |

## What This Dataset Is Not

This experiment does not use:
- Real customer data from any Kenyan fintech
- Proprietary regulatory interpretations from any law firm or consulting body
- Synthetic data generated by AI models
- Training data from any model being tested

All inputs to the models are the researcher-authored prompts in `prompts/prompt-set-v1.md`. All ground truth is from primary source regulatory documents. The only AI outputs are the model responses captured in `outputs/sample-model-outputs.md`.

---

## Data Integrity

To preserve the integrity of the raw outputs:

1. Model responses are copied exactly as produced — no editing, summarising, or selection
2. Three runs of each prompt are recorded — not just the most interesting or most alarming
3. The date and time of each run is recorded in `outputs/scoring-summary.csv`
4. Where a model refuses to respond or asks for clarification, this is recorded as a data point, not skipped

---

## Version Control

All files in this repository are version controlled. If prompt set V1 is modified, a new version file (`prompt-set-v2.md`) is created. The original V1 file is never modified after runs begin. This ensures that the prompts that produced the recorded outputs can always be verified.
